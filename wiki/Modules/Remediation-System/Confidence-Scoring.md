# Confidence Scoring

## Purpose

Confidence Scoring estimates the reliability of AI-generated patches. Each remediation task includes a confidence value that helps the developer or automation decide whether to apply the fix directly, review it manually, or discard it.

## Inputs

- LLM response containing a self-reported confidence score
- Error context complexity (used implicitly by the LLM)

## Processing

### Current Implementation

The confidence score is generated by the LLM as part of the remediation response. The system prompt instructs the LLM to:

1. Evaluate how certain it is about the root cause
2. Assess whether the proposed diff fully addresses the problem
3. Report a confidence value between 0.0 and 1.0

The parser extracts this value from the LLM response text.

### Score Interpretation

| Range | Interpretation |
|-------|---------------|
| 0.9–1.0 | High confidence — likely correct, candidate for auto-merge |
| 0.7–0.89 | Moderate confidence — review recommended |
| 0.5–0.69 | Low confidence — manual review required |
| 0.0–0.49 | Very low confidence — fix may be incorrect or incomplete |

### Planned: Threshold-Based Automation

A configurable threshold will determine automatic actions:

- Above threshold → auto-create PR, optionally auto-merge
- Below threshold → flag for human review
- Below minimum → mark as failed, do not create PR

## Outputs

- Confidence score (0.0–1.0) stored in `rem_confidence`
- Used by the Approval Workflow to determine next action
