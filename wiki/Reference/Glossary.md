# Glossary

Quick reference for important SoloDev terminology. For full definitions with context, see [Terminology](../Core-Concepts/Terminology).

| Term | Definition |
|------|-----------|
| AI Worker | Background service that polls pending remediation tasks, sends to LLM, stores patches |
| Auto-Pipeline | Module that detects tech stack and generates CI/CD YAML |
| Compound Tool | MCP tool orchestrating multiple controllers in a single workflow |
| Confidence Score | 0.0–1.0 value estimating AI patch reliability |
| Enforcement Mode | Solo Gate setting: strict, balanced, or prototype |
| Error Bridge | Service creating remediation tasks from errors and pipeline failures |
| Error Group | Collection of similar errors grouped by fingerprint |
| Finding | Security issue detected by the Scanner |
| Gitspaces | Cloud-based developer environments |
| Health Check | Configured HTTP endpoint monitor |
| Layer 1 | Core platform inherited from Gitness |
| Layer 2 | Observability modules added by SoloDev |
| Layer 3 | Intelligence modules added by SoloDev |
| Layer 4 | UI and CLI |
| MCP | Model Context Protocol — standard for AI agent tool access |
| Module | Discrete functional unit of the platform |
| Patch Diff | Unified diff generated by the AI Worker |
| Remediation | Structured task representing a failure and its proposed fix |
| Signal | Platform event indicating something needs attention |
| Solo Gate | Enforcement engine for quality and security policies |
| SoloDev Loop | Core feedback cycle: Detect → Analyze → Propose → Validate → Apply |
| Space | Organizational container (like a GitHub organization) |
| Tech Debt Item | Tracked issue representing deferred work |
